Checklist for Engineer MA Service

No.StageItemsCheck StandardAcknowledge or notRemarks
1Before service Ticket information Study and understand the fault description and suggested onsite operation in portal before proceed with the maintanance activity
Device information Check that the server SN is consistent with the ticket
Marking information Turn on UID to confirm server to be repaired；（Five elements of machine positioning: machine rack position, U position, SN, maintenance label, UID light）
2During service Shut down and restart Double check the marks and SN before shutdown and restart after getting permission
Take extra cautions on window ticket. If the server doesn't shutdown during the window maintanance time,please do not proceed with the repair acitivty and escalate inmediately.
Aware on surrounding and peripheral Take extra cautious on server status, rack status, railkit status and etc
When server have to be unmounted, be very careful in unmounting the server.
Updating Firmware Record the firmware version and FRU before perform any activities, make sure these information are flashed accordingly after service.
Confirm the device is general or customized, confirm the BIOS, BMC, FRU, SAS card, network card and other refresh requirements, then refresh as required
If server are power off during repair, collect necessary logs and keep for at least 3 months.
Check the FW version and customized information , ensure they are consistent with requirements after firmware refresh
Replacement of spare parts Make sure the accurate fault slot information and SN（especially Bad Hard Disk SN) before replacing spare part, if onsite found unmatch error with error provided in ticket, escalate immediately.
Confirm the spare part is physically good and don’t have any external damage
Before replacing the spare parts, it is necessary to have the spare parts PN information and spare parts capacity verified by the service manager or the designated person, and for different PN spare parts, the compatibility needs to be confirmed before the replacement operation can be carried out.
For non-multibrand spare parts, the good and bad parts must labelled with sticker.
All the parts are being dismantle and revert accordingly.
3After service Maintenance checks Confirm all the parts of the equipment are identified normally and the server indicator light is normal
Confirm that the external cables such as network cable and optical fiber are installed correctly and the network cable indicator is normal
Confirm the specification configuration provided by customer is completed.
Confirm that the server is mounted properly and securely. Make sure that the server is mounted to the rail kit. If there is no railkit, please escalate to the team.
Customer confirmation Confirm the customer feedback that the server is working normal and boot to OS normally
Confirm to return all non-returnable parts/access card to customer accordingly.
Remarks:
  1. The operation process should not violate the regulations of the hand-written "Red line operations"
  2. The operation process should not violate the regulations of the datacenter and customer requirements
  3. Select "no-operation" in the "complete or not" column if the relevant operation is not performed, fill in the reason in the "remarks" column if "no" is selected for any item
  4. After the ticket is completed, the engineer needs to submit the PDF of the inspection checklist signed by himself or the photos for centralized filing.
